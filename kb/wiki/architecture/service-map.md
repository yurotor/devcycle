# Service Map

How the repositories in this system relate to each other.

- **[COS.Lending.Selling.WebApi](../repos/cos-lending-selling-webapi.md)** — Core backend service implementing all business logic for loan purchasing, interest accrual, fee collection, volume fee processing, loan grooming, and transfer management. Contains F# business logic with C# infrastructure. Publishes events to outbox tables.
  - Communicates with: [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [Cos.Lending.Selling.Contracts](../repos/cos-lending-selling-contracts.md), [COS.Lending.Selling.Hooks](../repos/cos-lending-selling-hooks.md), [COS (Core Operating System)](../repos/cos-core-operating-system.md), [CRB.CosLending.Accounting.Api](../repos/crb-coslending-accounting-api.md), [AWS S3](../repos/aws-s3.md)
- **[COS.Lending.Selling.UI](../repos/cos-lending-selling-ui.md)** — React-based frontend providing user interfaces for loan inventory management, loan history viewing, pending approvals, and AI-powered chat queries. Implements role-based access control.
  - Communicates with: [COS.Lending.Selling.WebApi](../repos/cos-lending-selling-webapi.md), [cos-lending-selling-ai](../repos/cos-lending-selling-ai.md), [CRB Identity Provider](../repos/crb-identity-provider.md), [CRB Menu Service](../repos/crb-menu-service.md)
- **[COS.Lending.Selling.Hooks](../repos/cos-lending-selling-hooks.md)** — Notification integration service that converts domain events from the Selling system into standardized hooks/notifications published to the CosLending Hooks Hub for consumption by other systems.
  - Communicates with: [CRB.CosLending.Hooks.Hub.Service](../repos/crb-coslending-hooks-hub-service.md), [CRB Identity Provider](../repos/crb-identity-provider.md), [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md)
- **[Cos.Lending.Selling.Contracts](../repos/cos-lending-selling-contracts.md)** — Shared F# contracts library defining data types, models, and domain entities used across the system. No runtime dependencies - consumed by other repos at build time.
- **[Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md)** — Database model and Entity Framework Core context defining the PostgreSQL schema including entities, relationships, and 9 outbox tables for reliable messaging. Consumed by WebApi and data services.
- **[cos-lending-selling-ai](../repos/cos-lending-selling-ai.md)** — Python FastAPI service providing natural language query capabilities over loan data using AWS Bedrock (Claude). Manages chat sessions in DynamoDB and executes generated SQL against Selling DB.
  - Communicates with: [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [AWS Bedrock](../repos/aws-bedrock.md), [AWS DynamoDB](../repos/aws-dynamodb.md), [CRB Identity Provider](../repos/crb-identity-provider.md)
- **[cos-lending-selling-dags](../repos/cos-lending-selling-dags.md)** — Airflow orchestration layer running on AWS MWAA. Schedules and coordinates data ingestion, dbt transformations, daily interest calculations, volume fee processing, and reporting jobs.
  - Communicates with: [cos-lending-selling-ingestion](../repos/cos-lending-selling-ingestion.md), [cos-lending-selling-datatools](../repos/cos-lending-selling-datatools.md), [COS.Lending.Selling.WebApi](../repos/cos-lending-selling-webapi.md), [AWS ECS](../repos/aws-ecs.md), [AWS S3](../repos/aws-s3.md), [AWS Athena](../repos/aws-athena.md), [Vampire System](../repos/vampire-system.md)
- **[cos-lending-selling-data-utils](../repos/cos-lending-selling-data-utils.md)** — Account resolution utility that queries COS Lending Accounting service and loan operations database to map loans to proper financial accounts. Updates Selling DB with account mappings.
  - Communicates with: [CRB.CosLending.Accounting.Api](../repos/crb-coslending-accounting-api.md), [MPLConsumerLoansOperations Database](../repos/mplconsumerloansoperations-database.md), [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [CRB Identity Provider](../repos/crb-identity-provider.md)
- **[cos-lending-selling-datatools](../repos/cos-lending-selling-datatools.md)** — Data engineering repository containing dbt models for data transformation and analytics. Provides curated views of loan, contract, fee, and volume data for reporting and BI tools.
  - Communicates with: [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [AWS S3](../repos/aws-s3.md), [AWS Secrets Manager](../repos/aws-secrets-manager.md)
- **[cos-lending-selling-e2e-tests](../repos/cos-lending-selling-e2e-tests.md)** — End-to-end testing infrastructure simulating the entire lending environment with mock services (COS simulator, hooks, accounts). Validates loan processing from origination through purchase.
  - Communicates with: [COS.Lending.Selling.WebApi](../repos/cos-lending-selling-webapi.md), [cos-lending-selling-data-utils](../repos/cos-lending-selling-data-utils.md), [Arix Database](../repos/arix-database.md), [AWS S3](../repos/aws-s3.md), [TestRail](../repos/testrail.md)
- **[cos-lending-selling-ingestion](../repos/cos-lending-selling-ingestion.md)** — Python data ingestion service that syncs loan data from Arix, imports servicing CSV files from S3, fetches SOFR rates, bootstraps contract data, and publishes CloudWatch metrics.
  - Communicates with: [Arix Database](../repos/arix-database.md), [Vampire Database](../repos/vampire-database.md), [Contracts Database](../repos/contracts-database.md), [Volume Database](../repos/volume-database.md), [AWS S3](../repos/aws-s3.md), [Federal Reserve API](../repos/federal-reserve-api.md), [CloudWatch](../repos/cloudwatch.md), [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md)
- **[iac-cos-lending-selling](../repos/iac-cos-lending-selling.md)** — Infrastructure as code using Terragrunt/Terraform to provision and manage AWS resources including Aurora PostgreSQL, MWAA (Airflow), ECS Fargate, Lambda functions, S3 buckets, and monitoring.
