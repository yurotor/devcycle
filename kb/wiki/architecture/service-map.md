# Service Map

How the repositories in this system relate to each other.

- **[COS.Lending.Selling.WebApi](../repos/cos-lending-selling-webapi.md)** — Core business logic and transaction orchestration service. Handles loan purchases, interest accruals, fee processing, batch operations, and transfer management. Implements outbox pattern for reliable event publishing.
  - Communicates with: [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [Cos.Lending.Selling.Contracts](../repos/cos-lending-selling-contracts.md), [COS.Lending.Selling.Hooks](../repos/cos-lending-selling-hooks.md), [COS Transaction Service](../repos/cos-transaction-service.md), [COS Lending Accounting Service](../repos/cos-lending-accounting-service.md), [COS Storage Service](../repos/cos-storage-service.md), [AWS SQS](../repos/aws-sqs.md)
- **[COS.Lending.Selling.UI](../repos/cos-lending-selling-ui.md)** — React-based frontend providing loan inventory management, search/filtering, approval workflows, and AI-powered chat interface for loan queries. Primary user interface for MPLs, banks, and internal users.
  - Communicates with: [COS.Lending.Selling.WebApi](../repos/cos-lending-selling-webapi.md), [CRB Identity Provider](../repos/crb-identity-provider.md), [CRB AI Service](../repos/crb-ai-service.md), [CRB Menu Service](../repos/crb-menu-service.md)
- **[COS.Lending.Selling.Hooks](../repos/cos-lending-selling-hooks.md)** — Event notification service that converts domain events into standardized hooks notifications. Publishes loan status changes, batch completions, investor changes, fee charges, and overdue alerts to downstream consumers via NServiceBus.
  - Communicates with: [CRB.CosLending.Hooks.Hub.Service](../repos/crb-coslending-hooks-hub-service.md), [OAuth Identity Provider](../repos/oauth-identity-provider.md)
- **[cos-lending-selling-ai](../repos/cos-lending-selling-ai.md)** — AI-powered question-answering service using Claude via AWS Bedrock. Enables natural language queries about loan history, interest calculations, and transfers. Generates SQL from user questions and synthesizes human-readable responses.
  - Communicates with: [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [AWS Bedrock](../repos/aws-bedrock.md), [AWS DynamoDB](../repos/aws-dynamodb.md), [Identity Server](../repos/identity-server.md)
- **[cos-lending-selling-ingestion](../repos/cos-lending-selling-ingestion.md)** — Data ingestion and synchronization service that imports loan servicing data from S3, SOFR rates from Federal Reserve, and synchronizes loan status between Arix and Selling databases. Publishes CloudWatch metrics.
  - Communicates with: [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [AWS S3](../repos/aws-s3.md), [Federal Reserve API](../repos/federal-reserve-api.md), [Arix Database](../repos/arix-database.md), [Contracts Database](../repos/contracts-database.md), [Volume Database](../repos/volume-database.md), [Vampire Database](../repos/vampire-database.md), [AWS CloudWatch](../repos/aws-cloudwatch.md)
- **[cos-lending-selling-dags](../repos/cos-lending-selling-dags.md)** — Airflow-based orchestration service managing ETL workflows for data ingestion, transformation, curation, interest accruals, fee processing, and reporting. Schedules and monitors data pipeline execution.
  - Communicates with: [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [cos-lending-selling-datatools](../repos/cos-lending-selling-datatools.md), [cos-lending-selling-ingestion](../repos/cos-lending-selling-ingestion.md), [AWS ECS](../repos/aws-ecs.md), [AWS Athena](../repos/aws-athena.md), [AWS S3](../repos/aws-s3.md), [External loan data APIs](../repos/external-loan-data-apis.md), [SOFR data source](../repos/sofr-data-source.md), [Vampire System](../repos/vampire-system.md)
- **[cos-lending-selling-datatools](../repos/cos-lending-selling-datatools.md)** — Data engineering toolkit providing DBT transformations, data exports, and analytics support. Transforms raw ingested data into curated datasets for reporting and business intelligence.
  - Communicates with: [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [AWS S3](../repos/aws-s3.md), [AWS ECS](../repos/aws-ecs.md), [AWS Secrets Manager](../repos/aws-secrets-manager.md)
- **[cos-lending-selling-data-utils](../repos/cos-lending-selling-data-utils.md)** — Account resolution utility that maps loans to appropriate GL accounts by querying the Accounting API and applying custom mapping rules. Runs as CLI tool or scheduled job.
  - Communicates with: [CRB.CosLending.Accounting.Api](../repos/crb-coslending-accounting-api.md), [MPLConsumerLoansOperations Database](../repos/mplconsumerloansoperations-database.md), [Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md), [OAuth Authorization Server](../repos/oauth-authorization-server.md)
- **[Cos.Lending.Selling.DbModel](../repos/cos-lending-selling-dbmodel.md)** — Shared database model and Entity Framework context defining schema, entities, relationships, and outbox tables. Provides persistence layer for all services accessing the Selling PostgreSQL database.
  - Communicates with: [PostgreSQL Database](../repos/postgresql-database.md)
- **[Cos.Lending.Selling.Contracts](../repos/cos-lending-selling-contracts.md)** — Shared F# library defining canonical data types and contracts for loans, accounts, batches, invoices, receipts, and discrepancies. Ensures type safety and consistent domain modeling across all services.
- **[cos-lending-selling-e2e-tests](../repos/cos-lending-selling-e2e-tests.md)** — End-to-end testing infrastructure simulating the complete lending environment. Validates loan processing from origination through purchase, including interest, fees, and transfers. Includes mock COS services.
  - Communicates with: [COS.Lending.Selling.WebApi](../repos/cos-lending-selling-webapi.md), [cos-lending-selling-data-utils](../repos/cos-lending-selling-data-utils.md), [cos-lending-selling-datatools](../repos/cos-lending-selling-datatools.md), [AWS S3](../repos/aws-s3.md), [RabbitMQ](../repos/rabbitmq.md), [PostgreSQL](../repos/postgresql.md), [MSSQL](../repos/mssql.md)
